# Notebook Navigator

**GitHub:** https://github.com/[obsidian-notebook-navigator]

## What is the Notebook Navigator Plugin?

**Notebook Navigator** enhances **navigation and exploration** in Obsidian, making it easier to move between notes and understand your vault's structure. It lets you:

- 🗺️ Visualize your vault's folder hierarchy and relationships
- 🔍 Navigate between parent, sibling, and child notes easily
- 📍 Breadcrumb navigation showing where you are in the vault
- ⬅️ Jump to related notes with visual guides
- 📚 Browse folder contents without opening file explorer
- 🔗 Understand connections between different sections
- ✨ Quick-jump to frequently used notes and folders
- 📊 Overview of your entire vault structure

## Why Use Notebook Navigator?

**Notebook Navigator transforms how you explore your vault:**

- **Get context** — Always know where you are in your vault structure
- **Navigate faster** — Quick jumps to related notes
- **Reduce friction** — Stop hunting through folders
- **Visualize relationships** — See connections between sections
- **Improve discoverability** — Find notes you forgot you had
- **Better organization** — Understand your vault's structure deeply
- **Workflow optimization** — Move between related ideas faster
- **Spatial memory** — Visual navigation helps you remember locations

## How Notebook Navigator Works

1. **View breadcrumb** — See your current location in the vault
2. **Click navigation** — Jump between parent, sibling, or child notes
3. **Explore folder** — View folder contents without leaving note
4. **Jump quickly** — Bookmark frequent destinations
5. **Maintain context** — Always know where you are and what's nearby

## Example: Navigation in Action

**You're reading:** `Projects/2026/Project Alpha/Technical Notes.md`

**Breadcrumb shows:**
```
Projects > 2026 > Project Alpha > Technical Notes
   ↑       ↑       ↑              ↑
   |       |       |              └── Current note
   |       |       └── Can click to see all project files
   |       └── Can click to see all 2026 projects
   └── Can click to jump to Projects folder
```

**Sidebar shows:**
```
📁 Project Alpha (Current Folder)
  ├── 📄 Project Brief
  ├── 📄 Timeline & Milestones
  ├── 📄 Technical Notes ⭐ (You are here)
  ├── 📄 Design System
  ├── 📄 Team Notes
  └── 📄 Retrospective
```

**Quick actions:**
- Click breadcrumb to jump to any parent level
- Click folder to see all notes
- Click sibling to jump to related note
- Right-click to bookmark for quick access

## Navigation Patterns

### Pattern 1: Hierarchical Browsing
```
Start at broad category
↓
Drill down through subcategories
↓
Arrive at specific note
↓
Navigate sideways to related notes
```

**Use case:** Learning paths, research, documentation

### Pattern 2: Breadcrumb Quick-Jump
```
Deep in folder: A/B/C/D/E/F/note.md
↓
Click on "C" in breadcrumb
↓
Jump directly to that section
```

**Use case:** Reorienting when deep in nested folders

### Pattern 3: Sibling Navigation
```
Currently reading: Product Strategy
↓
See other notes in folder:
  - Market Analysis
  - Competitive Overview
  - Roadmap
↓
Click to read related note
```

**Use case:** Exploring related topics, context switching

### Pattern 4: Bookmarked Quick Access
```
Save frequent destinations:
  📌 Daily Notes
  📌 My Projects
  📌 Reference Library
↓
One-click access anytime
```

**Use case:** Daily workflow, quick jumps

## Tips for Effective Navigation

💡 **Keep folder depth reasonable** — 3-4 levels deep is optimal
💡 **Use descriptive folder names** — Makes breadcrumbs useful
💡 **Bookmark frequent paths** — Save time on recurring jumps
💡 **Understand your hierarchy** — Mental map helps navigation
💡 **Use consistent naming** — Alphabetical order in folders aids scanning
💡 **Review structure regularly** — Refine as your vault grows
💡 **Leverage breadcrumbs** — Better than hunting through folders
💡 **Create overview notes** — MOCs (Maps of Content) guide navigation

## Folder Structure Best Practices

**Good structure (enables navigation):**
```
30.Areas/
  ├─ Finance/
  ├─ Health/
  └─ Writing/

40.Projects/
  ├─ Project Alpha/
  │   ├─ Overview
  │   └─ Technical Design
  └─ Project Beta/

20.Daily/
  ├─ 2026/
  │   ├─ January/
  │   └─ February/
```

**Less helpful (hinders navigation):**
```
Files scattered at root
Too many nested levels (6+)
Unclear folder names
No consistent structure
```
